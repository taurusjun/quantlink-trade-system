## MODIFIED Requirements

### Requirement: Order status to ResponseType mapping
The counter_bridge `OnBrokerOrderCallback` SHALL map plugin OrderStatus to hftbase ResponseType as follows:
- ACCEPTED/SUBMITTED → NEW_ORDER_CONFIRM
- CANCELED → CANCEL_ORDER_CONFIRM
- REJECTED/ERROR → ORDER_ERROR
- PARTIAL_FILLED/FILLED → No response sent (log only, return immediately). TRADE_CONFIRM is generated by OnBrokerTradeCallback instead.
- UNKNOWN/SUBMITTING → Filtered (not forwarded)

#### Scenario: FILLED status does not generate TRADE_CONFIRM
- **WHEN** OnBrokerOrderCallback receives order_info with status=FILLED
- **THEN** counter_bridge logs the event but does NOT write any ResponseMsg to the SHM queue
- **AND** g_stats.filled_orders is incremented

#### Scenario: PARTIAL_FILLED status does not generate TRADE_CONFIRM
- **WHEN** OnBrokerOrderCallback receives order_info with status=PARTIAL_FILLED
- **THEN** counter_bridge logs the event but does NOT write any ResponseMsg to the SHM queue

#### Scenario: ACCEPTED still generates NEW_ORDER_CONFIRM
- **WHEN** OnBrokerOrderCallback receives order_info with status=ACCEPTED
- **THEN** counter_bridge writes a NEW_ORDER_CONFIRM ResponseMsg to the SHM queue (unchanged behavior)

#### Scenario: CANCELED still generates CANCEL_ORDER_CONFIRM
- **WHEN** OnBrokerOrderCallback receives order_info with status=CANCELED
- **THEN** counter_bridge writes a CANCEL_ORDER_CONFIRM ResponseMsg to the SHM queue (unchanged behavior)
