# 移除 CTP 行情调试打印 — 设计

## 上下文

`ctp_md_plugin.cpp` 的 `OnRtnDepthMarketData()` 是每个 tick 都会调用的回调函数。当前在行情转换和入队之前，有一段硬编码的 ag2603 调试打印代码。

## 目标 / 非目标

**目标：**
- 移除热路径中的调试代码，消除不必要的 `std::string` 构造和 stdout I/O

**非目标：**
- 不引入新的日志框架或条件编译宏（这是更大范围的重构）
- 不修改其他调试日志（如队列满的警告，那些是有用的运维日志）

## 决策

### 决策 1：直接删除，不替换为条件日志

直接删除整个 `if` 代码块（第 315-323 行），而不是用 `#ifdef DEBUG` 或日志级别守卫来替换。

**理由：** 这段代码是针对特定合约 `ag2603` 的临时调试代码，没有长期保留价值。如果未来需要调试行情数据，应该通过独立的调试工具或 NATS 订阅来实现，而不是在热路径中加入条件分支。
