# 如何更新任务 - 简明指南

**最后更新**: 2026-01-26

---

## 📋 任务文件位置

```
TASKS.md          ← 主任务跟踪文件（唯一，持续更新）
```

**不需要创建新文件**，所有任务状态更新都在这一个文件中进行。

---

## 🚀 最简单的方式：直接告诉我

### 方式1: 口头告诉我（推荐） ⭐

**开始任务**:
```
"开始任务#1"
"开始CTP行情接入，负责人是张三"
"启动任务#3和#4，预计下周完成"
```

**更新进度**:
```
"任务#1进度50%，已完成API集成"
"任务#3完成了5个指标"
"任务#2的CTP连接部分已完成"
```

**完成任务**:
```
"任务#1完成了"
"CTP行情接入已经完成并测试通过"
"任务#5 VWAP策略开发完成"
```

**报告问题**:
```
"任务#2遇到问题：CTP连接超时"
"任务#4被阻塞，等待第三方库"
```

**我会自动**:
- ✅ 更新 TASKS.md 文件
- ✅ 更新任务状态
- ✅ 记录时间和备注
- ✅ 更新进度统计
- ✅ 告诉你下一步应该做什么

---

## 📝 方式2: 自己编辑文件

### 步骤

1. **打开文件**:
   ```bash
   vim TASKS.md
   # 或
   code TASKS.md
   ```

2. **找到对应任务**，例如任务#1：
   ```markdown
   ### 任务 #1: CTP行情接入开发 🔴
   - **状态**: ⏸️ Pending        ← 修改这里
   - **负责人**: 待分配          ← 修改这里
   - **开始日期**: -             ← 修改这里
   - **预计完成**: -             ← 修改这里
   ```

3. **修改状态**:
   ```markdown
   - **状态**: 🔄 In Progress    ← 改为进行中
   - **负责人**: 张三             ← 填写负责人
   - **开始日期**: 2026-01-27    ← 填写开始日期
   - **预计完成**: 2026-02-03    ← 填写预计完成
   ```

4. **更新进度**:
   ```markdown
   - **备注**:
     - 2026-01-27: 已完成环境配置
     - 2026-01-28: 完成API集成，进度50%
   ```

5. **提交更新**:
   ```bash
   git add TASKS.md
   git commit -m "update: 任务#1进度更新"
   git push
   ```

---

## 🎯 状态定义

| 状态符号 | 含义 | 何时使用 |
|---------|------|---------|
| ⏸️ Pending | 待开始 | 任务还未开始 |
| 🔄 In Progress | 进行中 | 任务已开始，正在进行 |
| ✅ Completed | 已完成 | 任务完成并验收通过 |
| ⚠️ Blocked | 被阻塞 | 任务遇到阻碍，无法继续 |
| ❌ Cancelled | 已取消 | 任务不再需要 |

## 🎯 常见问题

### Q: 需要每次都提交Git吗？
**A**: 不需要。可以：
- 让我批量更新后一起提交
- 自己手动编辑后提交
- 每周或每个任务完成时提交一次

### Q: 任务状态更新会创建新文件吗？
**A**: 不会。永远只更新同一个 `TASKS.md` 文件。

### Q: 如果忘记更新怎么办？
**A**: 没关系，随时补充：
```
"补充：上周任务#1其实已经完成了50%"
```

### Q: 可以同时更新多个任务吗？
**A**: 可以：
```
"任务#1进度60%，任务#3进度30%，任务#4刚开始"
```
