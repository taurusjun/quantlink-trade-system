#### 数据链路详细说明

**1. 行情数据链路（模拟）**

```
步骤1: md_simulator 生成模拟行情
├─ 频率: 100 ticks/秒 (可配置)
├─ 算法: 随机游走 (Random Walk)
├─ 数据: Bid/Ask双边报价
└─ 输出: MarketDataRaw → 共享内存 md_queue

步骤2: md_gateway 读取并转发
├─ 从共享内存读取: SPSCQueue<MarketDataRaw>
├─ 转换为 protobuf: MarketData
├─ 发布到NATS主题: md.{exchange}.{symbol}
│   └─ 例如: md.SHFE.ag2603, md.SHFE.ag2605
└─ 延迟: < 1ms (本地共享内存)

步骤3: trader 订阅NATS行情
├─ 订阅通配符: md.*.ag2603, md.*.ag2605
├─ 回调处理: StrategyEngine.OnMarketData()
├─ 数据解析: protobuf → MarketData结构体
└─ 传递给策略: PairwiseArbStrategy

步骤4: 策略处理行情
├─ 更新价格窗口 (lookback_period: 100)
├─ 计算相关系数 (correlation)
├─ 计算价差 (spread) 和 Z-Score
├─ 判断入场/出场信号
└─ 生成订单 (当 |z| > entry_zscore)
```

**2. 订单发送链路（模拟）**

```
步骤1: 策略生成订单
├─ PairwiseArbStrategy.OnMarketData()
├─ 触发条件: |z-score| > 0.5
├─ 生成2腿订单:
│   ├─ Leg1: ag2603 BUY/SELL 10手
│   └─ Leg2: ag2605 SELL/BUY 10手
└─ 调用: StrategyEngine.SendOrder()

步骤2: trader 路由订单
├─ StrategyEngine.SendOrder()
├─ 构造 OrderRequest (protobuf)
├─ gRPC调用: ORSGateway.SendOrder()
├─ 地址: localhost:50052
└─ 超时: 5秒

步骤3: ors_gateway 接收订单
├─ gRPC服务端接收
├─ protobuf → OrderRequestRaw
├─ 写入共享内存: ors_request队列
├─ 队列大小: 4096条
└─ 延迟: < 1ms

步骤4: counter_gateway 处理订单
├─ 从共享内存读取: ors_request
├─ 调用 SimulatedCounter.SendOrder()
├─ 模拟处理:
│   ├─ 立即返回 order_id: EX_{timestamp}_{sequence}
│   ├─ 延迟10ms后: 发送ACCEPTED回报
│   ├─ 延迟50ms后: 发送FILLED回报
│   └─ 2%概率: 发送REJECTED回报
└─ 订单存储: 内存中的orders_ map
```

**3. 订单回报链路（模拟）**

```
步骤1: counter_gateway 生成回报
├─ ACCEPTED回报 (10ms后):
│   ├─ 构造 OrderResponseRaw
│   ├─ 状态: ACCEPTED (enum = 3)
│   ├─ 填充字段: order_id, symbol, side, exchange
│   └─ 写入共享内存: ors_response队列
│
├─ FILLED回报 (50ms后):
│   ├─ 构造 OrderResponseRaw
│   ├─ 状态: FILLED (enum = 5)
│   ├─ 填充成交信息:
│   │   ├─ filled_qty: 10
│   │   ├─ price: 模拟成交价
│   │   ├─ avg_price: 平均价
│   │   └─ last_fill_qty: 10
│   └─ 写入共享内存: ors_response队列
│
└─ REJECTED回报 (2%概率):
    ├─ 状态: REJECTED (enum = 8)
    ├─ 错误码: error_code
    └─ 错误信息: error_msg

步骤2: ors_gateway 转发回报
├─ 从共享内存读取: ors_response队列
├─ 转换: OrderResponseRaw → protobuf OrderUpdate
├─ 状态枚举映射:
│   ├─ 3 → ACCEPTED
│   ├─ 5 → FILLED
│   └─ 8 → REJECTED
├─ 发布NATS: order.updates.{strategy_id}
└─ 延迟: < 1ms

步骤3: trader 接收回报
├─ 订阅NATS: order.updates.*
├─ 回调: StrategyEngine.OnOrderUpdate()
├─ 解析 protobuf: OrderUpdate
└─ 分发到策略: strategy.OnOrderUpdate()

步骤4: 策略更新持仓
├─ PairwiseArbStrategy.OnOrderUpdate()
├─ 判断状态: if status == FILLED
├─ 更新持仓:
│   ├─ BaseStrategy.UpdatePosition() (总持仓)
│   └─ 分腿更新:
│       ├─ if symbol == ag2603: leg1Position += qty
│       └─ if symbol == ag2605: leg2Position += qty
└─ 记录日志: Leg1/Leg2 position updated
```

**4. 实时推送链路**

```
步骤1: trader 收集数据
├─ WebSocketHub 定时器 (1秒)
├─ collectDashboardData():
│   ├─ collectMarketData() - 最新行情
│   ├─ collectOrders() - 所有订单 (Multi-Strategy模式)
│   ├─ collectPositions() - 持仓信息
│   └─ collectStrategyStatus() - 策略状态
└─ 构造JSON消息

步骤2: WebSocket广播
├─ 序列化JSON
├─ 广播到所有连接的客户端
├─ 端口: 9201
└─ 连接数: clients (当前=1)

步骤3: Dashboard接收
├─ WebSocket连接: ws://localhost:9201
├─ 接收JSON消息
├─ 解析并更新Vue状态
└─ 响应式渲染页面
```

### 2.3 实盘数据链路对比

**实盘环境变化**:

```
【行情数据链路 - 实盘】
CTP/期货柜台 → 行情网关 → NATS → trader → Strategy
   ↑ 真实交易所           ↑ 需实现CTP适配器
   
【订单发送链路 - 实盘】  
Strategy → trader → ors_gateway → CTP适配器 → 期货柜台 → 交易所
                                   ↑ 需实现       ↑ 真实柜台
                                   
【订单回报链路 - 实盘】
交易所 → 期货柜台 → CTP适配器 → ors_gateway → NATS → trader → Strategy
                     ↑ 需实现                          ↑ 真实回报
```

**关键差异**:

| 环节 | 模拟测试 | 实盘环境 |
|------|---------|---------|
| 行情来源 | md_simulator (随机游走) | CTP/期货柜台 (真实行情) |
| 行情延迟 | < 1ms (本地) | 10-100ms (网络) |
| 订单路由 | 共享内存 | CTP协议/期货柜台API |
| 成交确认 | 固定延迟50ms | 不确定（受市场影响） |
| 滑点 | 无 | 存在（市场波动） |
| 拒单逻辑 | 固定概率2% | 真实风控规则 |
| 网络稳定性 | 100% | 可能断线/重连 |
